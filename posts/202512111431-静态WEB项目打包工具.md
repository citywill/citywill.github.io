# 静态WEB项目打包工具

基于 Go 语言的静态程序打包工具。

开发静态程序（包括 HTML、CSS、JS、图片等文件）后，需要依赖 Web 中间件的安装和配置完成部署，这个过程较复杂，可以使用本工具将项目打包成可执行文件，交付时只需要将单个文件发送给用户，用户双击运行即可自动打开浏览器访问程序。

Github：[citywill/static-packer: 基于 Go 语言的静态程序打包工具](https://github.com/citywill/static-packer)

## 特性

- 方便部署：不用安装web中间件（nginx等），双击可执行。
- 易于传输：打包后资源大小只比原静态程序包大10m左右。
- 节约资源：程序基于浏览器访问，相比electron等框架，不会额外占用内存。
- 开发友好：静态web页面是最容易开发，也是AI辅助开发最友好的方案。无论是纯html开发，还是使用前端工程框架（需build），不需要移植开发，可直接打包。

## 使用方法

1. 将静态资源放置到`static`目录下，默认页面为`index.html`。

2. 编译：

```bash
# linux 下
go build -o dist/{app_name}
# windows 下
go build -ldflags "-H=windowsgui" -o dist/{app_name}.exe main.go
```

3. 运行`dist/{app_name}`(Windows 下`dist/{app_name}.exe`)

程序运行后，自动打开浏览器访问静态资源，同时在系统托盘创建图标及菜单，用于控制程序访问和退出。

## 配置

在可执行文件的同目录下，可创建`config.json`文件，用于配置程序名称和打开端口号，内容如下：

```json
{
  "app_name": " 程序名称",
  "port": "8080"
}
```
