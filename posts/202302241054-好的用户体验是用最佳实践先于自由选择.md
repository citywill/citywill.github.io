# 好的用户体验是用最佳实践先于自由选择

![](./assets/202302241054-好的用户体验是用最佳实践先于自由选择-蘸料.png)

昨天晚上去了一家人气很高的烤肉店。人气高到什么程度呢，这家店在长春市最热闹的步行街开了三家店，都叫一个名字。

**其实就是普通的韩式烤肉。装修也不高档，服务也一般般。但是，他们做对了一件事情：服务员会告知食客，蘸料要怎么用。比如肥牛蘸湿料，五花蘸干料，包肉要在此基础上蘸黄酱，等等。甚至干料要用一勺红料配两勺黄料，也会告诉我们。**

烤肉其实挺简单。只要肉新鲜，按照正确的吃法，味道都差不太多。但是我以前所有吃过的各种档次的烤肉店，都不会这么清晰以至于不容置疑的告知蘸料的用法。

我由此得到了这样几点感受：

1. 多数时候，好的用户体验式，是不给用户选择，而是给他们最佳实践方案。
2. 最佳实践降低了初级用户的试错成本和心智负担，也并没有完全阻断资深用户对高级用法的尝试。
3. 多数用户只有先成功的成为初级用户，才能升级到资深用户的。
4. 只有赢得初级用户体验，产品才能取得成功。因此产品要特别重视初级用户的应用体验。正如我们要通过提前构思媒体评价来设计产品特性一样，我们也要提前构思 quick start 来帮助我们提升用户体验。
5. 一般来说，quick start 越短越好。如果你的产品无法提供一个简短的快速入门，就说明你的产品设计无法给初学者来带友好的体验。
6. 对最佳实践的介绍，仅提供 quick start 是不够的，还应该包含一个覆盖全面又重点突出的用户指南。指南是一系列文档，最好以“如何”作为标题的开端。比如“如何吃五花肉”。
7. 在此基础上，用户才需要一个工具书式的手册，为自己自由探索产品保驾护航。

饮食是最复杂的消费产品门类，比人的饮食更复杂的是组织信息化和数字化系统。在《Don't Make Me Think》一书中，作者提出通过测试等方法减少用户使用过程中的不必要迟钝。但是用户体验绝不仅仅存在于交互设计和信息架构上，要想做好用户体验，必需将体验设计下延到接口设计、系统架构等多个方面。总之，无论多复杂的系统，都要先找到一个关键点，用最佳实践包装，再引导更复杂和强大的功能。
