# 使用 ChatGPT 辅助编程的效用提升和风险

![](../assets/202302232108-使用%20ChatGPT%20辅助编程的效用提升和风险.jpg)

最近一直在探索使用 ChatGPT 辅助工作，并在团队培训、推广，建议产品经理和程序员使用。目前看效果最明显的作用是辅助编程。使用 ChatGPT 的确可以帮助程序员快速地生成代码，提升编程效率。

这种效率提升主要体现在编码效率和质量方面。

ChatGPT 的确可以提升编程速度。比如我要求 ChatGPT 写一个百度新闻爬虫，几秒钟就搞定了，我又要求增加防反爬机制，也搞的头头是道。ChatGPT 写出来的爬虫，稍微调试修改，就可以挂到调度上了。

通过 ChatGPT 辅助开发的代码质量也会得到提高。以往演示评审会上，我提问题最多的往往不是核心算法，而是新手的接口命名、文档写作等问题。代码写的差点有时候问题不大，接口命名和文档等成果关系到团队价值输出，写不好是最麻烦的。用 ChatGPT 生成的代码，在规范性上好于平均水平，而且会直接生成可读性较好的文档。别看 ChatGPT 经常满嘴跑火车，但在遣词用句和八股套路上，比多数刚毕业的理工萌新要强。

由 ChatGPT 辅助提升编码效率和质量，程序员就可以把更多的时间放到领域设计和核心算法上了。

然而在应用 ChatGPT 解决眼前问题的同时，我们也要警惕长期使用的负面影响。

表面上，以编写 CURD 为主的程序员可能要面临转型了。另外，ChatGPT 不仅可以生成规范的代码模板，更能根据自然语言返回较为复杂的逻辑，这是目前的低代码开发平台所不具备的能力。所以以快速生成 CURD 代码为主要目标，并且背后没有类似 ChatGPT 生成算法赋能的低代码开发平台，也要面临失去竞争力的尴尬境地。

但最让我担心的，是一个隐约的可能性，那就是边际递减效应。

边际递减效应指的是，在一个系统内，随着自变量的持续增加，应变量由一开始的增加，逐渐转向减少的现象。换句话说，在一开始的时候增加投入可以带来很好的效果，但是当你持续增加投入时，效果会逐渐变弱，甚至反而会产生负面影响。

举个例子，当一个人投入更多的时间和精力学习某个主题时，一开始会有比较快的学习效果，但随着时间的增加，学习效果逐渐变得越来越慢，甚至到最后没有效果了。再比如，使用化肥可以让农作物产量提升，但是长期施肥会对土壤造成负面影响，导致作物的产量下降。这些都是边际效用递减的体现。

再可见的未来，软件研发企业都将普遍采购类 ChatGPT 的辅助编程 AI，这些 AI 生成的代码又会在一定程度上影响到 GitHub 上的代码多样性和创造性。ChatGPT 又使用 GitHub 的代码进行训练，长此以往，ChatGPT 生成的代码也会趋于平庸。

甚至我认为，边际递减已经在 AI 写作上实际发生了。你看今日头条、知乎上的口水文越来越多，质量越来越差，用这些文本训练模型再由 AIGC 生成出来，周而复始。如果不加以干预，边际递减终将发展成热寂，我们也终将被迫回到 PGC 的时代了。
